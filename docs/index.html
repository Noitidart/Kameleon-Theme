HTML Structure

HTML5

This template embeds a custom font into the webpage. To change which elements are replaced find the cufon entry in the functions.js file. By default only page headings are replaced via cufon.

This template uses the HTML5 Boilerplate framework for a rock solid base to support the template. I recommend watching this screencast by Paul Irish which walks you through the basic setup: http://net.tutsplus.com/tutorials/html-css-techniques/the-official-guide-to-html5-boilerplate/. Please note that not all the functions of the boilerplate have been used. Some redundant aspects have been removed for convince.

Whereas HTML5 brings a lot of new advanced features (geolocation, drag and drop, local storage etc) this template mainly utilizes the new markup. What's the difference? Well instead of writing a div with an id of nav or sidebar, we can just use their HTML5 counterparts instead. It reduces markup and feature-proofs your website.

Main Elements

The template is structured around 3 main elements. The header, page and footer. The header contains the logo and the navigation. The page contains the slider (index only), page title, page content and sidebar. The footer contains the copyright information (which you may change) and the sub-navigations.

Logo

This template uses a CSS technique to replace the H1 tag text with the logo image. This helps SEO. To change the logo find h1#logo in either the light or dark CSS files. (They are seperated as the template uses two version depending on the theme. Simply change the path and the dimensions.

Navigation

This template uses a CSS only navigation. This ensures maximum cross compatibility between browsers and is SEO friendly. The navigation functions from lists. The class "current" designates which page the user is on. To create a drop down simply create a new list within another one. For example:

Tooltips

Admina uses the excellent tipsy tooltips plug-in to show tooltips. I've built a couple of inbuilt classes to help you get the most configurability out of the plugin. To add a tooltip to something simply add the class "tooltip" and add the tooltip text in the title attribute of the tag. So take this for example:



You can configure how the tooltip shows more extensively by adding polar directions to class. So for example adding the class " tipsy-north" will make your tooltip appear above the object instead of the default below.

CSS

I'm using two CSS files in this theme. Style.css is a html5 reset/setup file. Many browser interpret the default behavior of html elements differently. By using a reset CSS file, we can work round this.

The second CSS file base.css contains all the visual styling for the theme. It is separated into section such as:

/*////   - Headings -   ////*/

.some-code {

}

/*////   - Misc -   ////*/

.some-code {

}

/*////   - Nav -   ////*/

.some-code {

}


Etc, etc.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.
